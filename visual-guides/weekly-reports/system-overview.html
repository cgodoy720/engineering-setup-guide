Weekly Reports System

system overview — automated cohort reports

The Weekly Reports system generates automated weekly reports for cohorts, including attendance overview, at-risk builders, activities completion, assessments, and AI-generated narrative insights using Claude Sonnet 4.5. Reports are delivered via HTML email with CSV attachments and optional Slack notifications. All endpoints require page:weekly_reports permission.

Report Content: Each weekly report includes attendance overview, at-risk builders (based on attendance thresholds), activities and completion rates, graded activities, assessments, and AI-generated narrative insights using Claude Sonnet 4.5 (currently hidden in email but generated). External cohorts are excluded from reports.
3 — Pages
Route Component Key Functionality
/admin/weekly-reports WeeklyReports Admin dashboard for managing weekly report configurations, Slack settings, recipients, viewing report logs, and manually triggering reports. Requires page:weekly_reports permission. Features include: config management (week start/end days, enable/disable), Slack channel selection and testing, recipient management (add/edit/remove), report log viewing with status and error messages, manual trigger for single cohort or all cohorts, and cohort lookup (active non-workshop cohorts only).
4 — API Routes
All endpoints require page:weekly_reports permission.
Config Management
  • GET /api/admin/weekly-reports/configs — Get all configs
  • PATCH /api/admin/weekly-reports/configs/:cohortId/toggle — Toggle report enabled
  • PATCH /api/admin/weekly-reports/configs/:cohortId — Update config (week_start_day, week_end_day, etc.)
Slack Management
  • GET /api/admin/weekly-reports/slack/channels — Get Slack channels
  • PATCH /api/admin/weekly-reports/configs/:cohortId/slack — Update Slack config
  • POST /api/admin/weekly-reports/slack/test — Test Slack notification
Recipients
  • GET /api/admin/weekly-reports/recipients/:cohortId — Get recipients
  • POST /api/admin/weekly-reports/recipients — Add recipient
  • PATCH /api/admin/weekly-reports/recipients/:id — Update recipient
  • DELETE /api/admin/weekly-reports/recipients/:id — Remove recipient
Report Logs
  • GET /api/admin/weekly-reports/logs/:cohortId — Get report logs for cohort
Manual Triggers
  • POST /api/admin/weekly-reports/trigger/:cohortId — Trigger report for single cohort
  • POST /api/admin/weekly-reports/trigger-all — Trigger reports for all enabled cohorts
Cohorts
  • GET /api/admin/weekly-reports/cohorts — Get active non-workshop cohorts
5 — Database Tables
3 tables support the weekly reports system. See schema.sql for complete DDL with indexes and constraints.
weekly_report_config
Per-cohort configuration for weekly reports. Includes week calculation settings (week_start_day 0-6, week_end_day 0-6), enable/disable flag, and Slack settings. UNIQUE constraint on cohort_id ensures one config per cohort.
  • id PK
  • cohort_id FK → cohort (UNIQUE)
  • week_start_day INTEGER (0-6, Sunday=0)
  • week_end_day INTEGER (0-6)
  • report_enabled BOOLEAN
  • slack_channel_id VARCHAR
  • slack_enabled BOOLEAN
weekly_report_recipients
Email recipients for weekly reports per cohort. Can be linked to a user account (user_id nullable) or just an email address. UNIQUE constraint on cohort_id + email ensures no duplicate recipients per cohort.
  • id PK
  • cohort_id FK → cohort
  • user_id FK → users (nullable)
  • email VARCHAR NOT NULL
  • name VARCHAR
  • active BOOLEAN
  • added_by FK → users
  • UNIQUE(cohort_id, email)
weekly_report_log
Logs of all report generations and deliveries. Stores report data as JSONB, tracks delivery status (pending/sent/failed), error messages, and delivery channels (email, slack) as JSONB object.
  • id PK
  • cohort_id FK → cohort
  • week_start_date DATE
  • week_end_date DATE
  • recipients_count INTEGER
  • report_data JSONB
  • status VARCHAR (pending/sent/failed)
  • error_message TEXT
  • delivery_channels JSONB ({email: bool, slack: bool})
  • created_at TIMESTAMP
