Payment & Financial System

system overview — architecture, pages, data flow

The Payment & Financial system manages user financial documentation, employment information, and invoice calculations. It provides both a User Portal for document uploads, employment info management, and financial planning, and an Admin Dashboard for staff to manage user documents and preview files. All documents are stored in Google Cloud Storage with signed URLs for secure access.

2 — Two Sides of the System
User Portal

User-facing portal for managing financial documents and employment information. Users can upload required documents, calculate invoices, manage employment details, and schedule financial planning sessions.

  • /payment Main portal dashboard
  • Document uploads (Good Job Agreement, Bill.com Guide, Bond FAQs, Employment Contract)
  • Invoice calculator (tiered/flat rates)
  • Employment info management (CRUD)
  • Financial planning scheduling (Calendly integration)
Admin Dashboard

Staff-only dashboard for managing user documents and employment information. Requires page:payment_admin permission. Admins can search users, upload documents on their behalf, and preview files.

  • /payment-admin Admin dashboard
  • User search and selection
  • Upload documents for users
  • Preview user documents
Public Pages
  • /payment-terms Static terms page with FAQs and definitions
3 — User Pages
Route Page Key Functionality Document Types
/payment Payment Main user portal with document uploads, invoice calculator, employment info management, and financial planning scheduling. Good Job Agreement, Bill.com Guide, Bond FAQs, Employment Contract
/payment-terms PaymentTerms Static terms page with FAQs and definitions. No authentication required. N/A
Document Upload Flow: Users can upload four types of documents: goodJobAgreement, billComGuide, bondFaqs, and employmentContract. Each user can have only one document per type (enforced by UNIQUE constraint on user_id + document_type). Files are stored in Google Cloud Storage with signed URLs for secure access.
4 — Admin Pages
Route Page Key Features
/payment-admin PaymentAdmin User search/selection, upload documents for users, preview documents. Requires page:payment_admin permission.
5 — API Layer
/api/payment/admin
  • POST /upload-document Upload document for user (requires page:payment_admin)
  • GET /documents/:userId Get all documents for a user
/api/payment
  • POST /upload-document Upload own document
  • GET /documents Get own documents
  • PUT /employment-info Update employment info
  • GET /employment-info Get employment info
Controller
  • paymentController Handles all payment routes
  • Admin routes: upload/get documents for users
  • User routes: upload/get documents, employment info CRUD
6 — Core Database Tables
2 tables support the payment system. Documents are stored in Google Cloud Storage with metadata tracked in the database. See schema.sql for complete DDL.
payment_documents
  • id Primary key
  • user_id Foreign key to users
  • document_type Enum: goodJobAgreement, billComGuide, bondFaqs, employmentContract
  • original_name Original filename
  • file_name Stored filename
  • file_path GCS path
  • file_url Signed URL (temporary)
  • uploaded_at Timestamp
  • UNIQUE: user_id + document_type
employment_info
  • id Primary key
  • user_id Foreign key (UNIQUE)
  • company_name Employer name
  • position Job title
  • start_date Employment start
  • salary Compensation
  • employment_type Enum: full-time, part-time, contract, freelance
  • status Enum: employed, unemployed, job-searching, self-employed
  • updated_at Last update timestamp
Storage: All document files are stored in Google Cloud Storage (GCS). The database stores metadata and file paths. Signed URLs are generated on-demand for secure access with expiration times. Files are organized by user_id and document_type.
