External Cohorts System

system overview — architecture, pages, data flow

The External Cohorts system manages enterprise cohorts and workshops for external organizations. It provides a unified signup flow using access codes, admin assignment, participant management, curriculum viewing, and conversation tracking. Supports three access levels: public signup (no auth), admin dashboard (requires page:external_cohorts), and cohort admin (requires page:cohort_admin).

4 — API Routes
Public (No Auth)
  • POST /api/external-cohorts/access Sign up with access code
  • GET /api/external-cohorts/validate-code/:code Validate access code
Admin (page:external_cohorts)
  • Full CRUD for cohorts
  • Admin assignment/listing/removal
  • Bulk invite participants
  • Resend invitations
  • Stats, participants, attendance
  • Curriculum viewing
  • Participant/task conversations
Cohort Admin & Internal
  • GET /api/external-cohorts/cohort-admin/my-cohorts (page:cohort_admin)
  • GET /api/external-cohorts/internal/list (CurriculumEditor)
  • POST/PUT /api/external-cohorts/internal (CurriculumEditor)
Permission Requirements: Admin routes require page:external_cohorts, cohort admin routes require page:cohort_admin, and internal routes are for CurriculumEditor integration.
5 — Core Database Tables
4 core tables support the External Cohorts system. The tables handle cohort management, admin assignments, invitations, and user roles. The users table is extended with cohort_id FK and enterprise roles. See schema.sql for complete DDL.
Cohort Management
  • cohort Cohort records with access codes, types, dates
  • cohort_admins Admin assignments to cohorts
Invitations & Users
  • cohort_invitations Participant invitations with status tracking
  • users Extended with cohort_id FK and enterprise roles
Table Key Columns Purpose
cohort cohort_id (UUID PK), name, start_date, end_date, cohort_type (builder/workshop/external), access_code (UNIQUE), description, contact_name, contact_email, is_active, course_id (FK) Cohort records with auto-generated access codes (format: EXT-{timestamp}-{random})
cohort_admins admin_id (PK), user_id (FK nullable), cohort_id (FK), pending_email, role Admin assignments to cohorts, supports pending admins via email
cohort_invitations invitation_id (PK), cohort_id (FK), email, status (pending/sent/registered/expired), sent_at, registered_at, user_id (FK nullable) Participant invitations with status tracking and registration linking
users (extended) cohort_id (FK), roles: enterprise_builder, enterprise_admin User accounts extended with cohort assignment and enterprise roles
