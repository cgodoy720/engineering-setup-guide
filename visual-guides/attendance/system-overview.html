Attendance System

system overview — architecture, pages, data flow

The Attendance system manages builder check-ins with camera capture, tracks daily attendance records with status indicators (present/late/absent/excused), provides admin dashboards for attendance management and cohort performance analytics, and handles excuse requests with approval workflows. It supports both builder self-check-in and staff/admin manual record management.

2 — Builder-Facing Pages
Route Page Key Functionality
/attendance-dashboard Builder Check-In Builder check-in interface with camera capture for photo verification. Shows today's overview with status indicators, cohort cards with attendance summary, search functionality to find builders, and real-time check-in status updates.
/attendance-login Staff Login Staff/admin login page for attendance system. Separate authentication from main builder login. Required for accessing admin attendance features.

6 — Core Database Tables
3 core tables support the attendance system. Builder attendance tracks daily check-ins, excused absences handle excuse requests and approvals, and volunteer attendance manages volunteer slot attendance. See schema.sql for complete DDL.
builder_attendance_new
  • attendance_id PRIMARY KEY
  • user_id FK to users
  • attendance_date Date of attendance
  • check_in_time Timestamp of check-in
  • photo_url Photo from check-in
  • late_arrival_minutes Minutes late
  • status present/late/absent/excused
  • notes Additional notes
  • UNIQUE: user_id + attendance_date
excused_absences
  • excuse_id PRIMARY KEY
  • attendance_id FK to attendance
  • user_id FK to users
  • absence_date Date of absence
  • excuse_reason Sick/Personal/Program Event/Technical Issue/Other
  • excuse_details Details text
  • staff_notes Staff notes
  • processed_by FK to users
  • status pending/approved/denied
  • UNIQUE: user_id + absence_date
volunteer_attendance
  • attendance_id PRIMARY KEY
  • slot_id FK to volunteer slots
  • volunteer_user_id FK to users
  • status pending/attended/no_show/cancelled/excused
  • checked_in_at Check-in timestamp
  • photo_url Photo URL
  • UNIQUE: slot_id + volunteer_user_id
