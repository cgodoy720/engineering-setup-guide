Admissions System

system overview — architecture, pages, data flow

The Admissions system manages the full lifecycle of applicant intake — from initial account creation through info sessions, application review, workshops, program acceptance, pledge completion, and onboarding. It has two distinct user-facing sides: an Applicant Portal (public, no auth required beyond applicant login) and an Admin Dashboard (staff-only, requires page:admissions permission).

2 — Two Sides of the System
Applicant Portal

Public-facing pages for people applying to the program. Uses a separate auth system (applicant login, not builder login). Routes are defined outside the main app shell.

  • /apply/signup Account creation
  • /apply Progress dashboard
  • /info-sessions Event registration
  • /application-form Multi-section form
  • /workshops Workshop registration
  • /pledge 6-step pledge wizard
  • /onboarding 8-step onboarding
Admin Dashboard

Staff-only dashboard for managing the entire admissions pipeline. Requires page:admissions permission. Tabbed interface with bulk actions and analytics.

  • /admissions-dashboard Main dashboard (6 tabs)
  • Overview — KPIs, demographics, stage breakdowns
  • Applicants — Filter, search, bulk actions
  • Info Sessions — Create/manage events
  • Workshops — Create/manage workshops
  • Leads — CSV import, source tracking
  • Emails — Automation, templates, tracking
3 — Applicant Pages
Route Page Key Functionality Connects To
/apply/signup ApplicantSignup Account creation (name, email, password). Email verification sent on submit. /apply
/apply ApplicantDashboard 4 progress cards (Info Session, Application, Workshop, Pledge/Onboarding). Status-driven UI shows locked/available/completed states. Cards dynamically swap based on stage. All applicant pages
/info-sessions InfoSessions Browse available info sessions. Register/cancel. Shows date, time, location, capacity. Online sessions show meeting link. /apply
/application-form ApplicationForm Multi-section form with auto-save (1s debounce). Eligibility checking. Conditional questions. Progress tracking. Address autocomplete. /apply
/workshops Workshops Browse workshops (only if invited). Register with laptop selection. Access window control. "Enter Workshop" converts to builder account. /apply, /dashboard
/pledge Pledge 6-step wizard: Introduction, Learning, Community, Building, Information, Signature. Canvas-based signature with touch support. /apply
/onboarding Onboarding 8-step task wizard: Program Details, Attendance Policy, Slack, Pursuit Email, Google Calendar, Kisi, Building in Public, Tech News. Creates builder account when all required tasks complete. /apply, /dashboard
4 — Admin Pages
Tab / Route Purpose Key Features
Overview Statistics and analytics KPI tiles, stage breakdowns, demographic charts, conversion funnels
Applicants Application management Filterable table (stage, status, score), bulk actions (invite, email, admit, reject), notes, deliberation flags (yes/maybe/no)
Info Sessions Event management Create/edit sessions, manage registrations, mark attendance (attended/late/no-show), capacity tracking
Workshops Workshop management Create/edit workshops, manage registrations, mark attendance, laptop tracking
Leads Lead pipeline CSV import, source tracking, conversion to applicant, lead status management (new/nurturing/qualified/converted)
Emails Email automation Automated email sequences, queue management, send history, email mapping (personal to Pursuit email)
/admissions-dashboard/applicant/:id Individual applicant detail Full application responses, AI analysis scores, workshop performance, event participation, email tracking, staff notes, deferral/deliberation controls

22 tables support the admissions system. The core tables below handle applicant accounts, applications, stages, events, and analysis. See database-schema.html for the full ER diagram and schema.sql for complete DDL.
Core Identity

  • applicant_notes Staff notes per applicant
  • applicant_email_mapping Personal ↔ Pursuit email
Application Form

  • section Form sections with ordering
  • question Conditional questions
  • choice_option Multiple choice values

Events & Registration

  • event_registration Registrations with attendance

Analysis & Leads

  • lead_engagement Interaction history
  • email_automation_log Email queue and tracking
