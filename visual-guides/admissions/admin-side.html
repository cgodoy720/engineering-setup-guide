Admin Side

admissions dashboard — tabs, features, and actions

The admin dashboard at /admissions-dashboard is a tabbed interface with 6 tabs. It requires page:admissions permission. Staff use it to manage the entire admissions pipeline — from reviewing applications and managing events to importing leads and running email automation.

1 — Overview Tab
Overview Default Tab
The landing view when entering the dashboard. Provides a high-level snapshot of the current admissions cycle with KPI tiles, demographic breakdowns, and stage distribution.
KPI Tiles
Total applicants, applications submitted, info sessions attended, workshops attended, accepted count, rejection rate
Demographics
Breakdowns by race, gender, education level, borough, age range. Visualized as charts.
Stage Distribution
How many applicants are at each stage of the pipeline. Funnel visualization.
Conversion Metrics
Lead → Applicant → Submitted → Reviewed → Admitted conversion rates
API: GET /api/admissions/dashboard/overview, GET /api/admissions/stats
2 — Applicants Tab
Applicants Primary Management
The core management view. A filterable, paginated table of all applicants with their current stage, application status, AI scores, and deliberation flags. Supports bulk actions for batch operations on selected applicants.
Filters
Filter by: current stage, program admission status, AI score range, deliberation flag (yes/maybe/no), cohort, search by name/email
Table Columns
Name, email, stage, admission status, overall score, recommendation, deliberation, submitted date, actions
Deliberation Flags
Staff can set yes, maybe, or no on each applicant. Used during review meetings.
Notes
Add/edit/delete staff notes per applicant via NotesModal. Notes link to the staff user who created them.
Bulk Actions — select multiple applicants and apply:
Invite to Workshop
Send Email
Admit
Reject
Waitlist
Update Stage
Defer
API: GET /api/admissions/applications, POST /api/admissions/bulk-actions
3 — Info Sessions Tab
Info Sessions Event Management
Create and manage info session events. View registrations, mark attendance, and track capacity.
  • Create/Edit — Title, date, time, location, capacity, online flag, meeting link
  • Registration List — View all registered applicants per session
  • Mark Attendance — Set status to attended, attended_late, very_late, no_show, cancelled
  • Manual Registration — Register an applicant manually via ManualRegistrationModal
  • Capacity Tracking — Visual indicator of spots filled vs. available
  • Cancel/Delete — Cancel registrations, delete sessions
API: GET|POST|PUT|DELETE /api/admissions/info-sessions, PUT /api/admissions/attendance/info-session/:eventId/:applicantId
4 — Workshops Tab
Workshops Event Management
Create and manage workshop events. Similar to info sessions but with additional workshop-specific features.
  • Create/Edit — Title, date, time, location, capacity, cohort assignment
  • Registration List — View registered applicants, laptop needs column
  • Mark Attendance — Same status options as info sessions
  • Laptop Tracking — See which attendees need a laptop provided
  • Workshop Type — admissions for the admissions pipeline workshops
API: GET|POST|PUT|DELETE /api/admissions/workshops, PUT /api/admissions/attendance/workshop/:eventId/:applicantId
5 — Leads Tab
Leads Pipeline
Manage pre-applicant leads. Import from CSV, track sources, monitor conversion to applicants.
CSV Import
Upload CSV files with lead data. Maps to lead and lead_source tables. Deduplicates by email.
Source Tracking
Multi-source per lead (events, referrals, website, outreach). Each source tracked with type, name, and capture date.
Status Pipeline
Statuses: new → nurturing → qualified → converted. Also: unsubscribed, invalid.
Conversion
When a lead creates an applicant account, the lead.applicant_id is linked and status becomes converted.
API: GET /api/admissions/leads, POST /api/admissions/leads/import, GET /api/admissions/leads/stats
6 — Emails Tab
Emails Automation
Automated email sequences triggered by applicant stage changes. Queue management, send history, and email mapping (personal email to Pursuit email).
Automation Engine
Stage-based email triggers. Tracks send count, next send date, and queue status via email_automation_log.
Queue Management
View queued emails, manually trigger automation runs. Each applicant gets one entry per email type (UNIQUE constraint).
Send History
Track when emails were sent, opened, and clicked. Historical log with timestamps.
Email Mapping
Map personal emails to Pursuit emails via applicant_email_mapping. Used during onboarding.
API: GET /api/admissions/email-automation/stats|queued|history, POST /api/admissions/email-automation/run
7 — Applicant Detail View
ApplicationDetail /admissions-dashboard/applicant/:id
A full-page detail view for a single applicant. Accessed by clicking an applicant row in the Applicants tab. Shows everything about the applicant in one place.
Header
Applicant name, email, current stage badge, admission status badge, "Back to Applicants" nav
AI Analysis Scores
Overall, Learning, Grit, Critical Thinking scores (0-100). Recommendation label. Final status (human override).
Application Responses
Collapsible sections: Eligibility, Key Analysis, other categories. Shows all questions and answers.
Workshop Performance
If grade data exists: task breakdowns with scores, conversation history from BigQuery integration.
Event Participation
Info sessions and workshops the applicant registered for and attended, with timestamps.
Staff Controls
Deliberation dropdown (yes/maybe/no), eligibility override, deferral status, actions modal for single-applicant operations.
Email Tracking
All automated emails sent to this applicant, with sent/opened/clicked timestamps.
Notes
Sidebar with all staff notes. Add/edit/delete. Each note shows author and timestamp.
API: GET /api/admissions/applicant/:applicantId/detail
