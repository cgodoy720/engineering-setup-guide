Admin & Management System

system overview — permissions, organizations, cohorts, prompts

The Admin & Management system provides comprehensive control over user permissions, organizational hierarchies, cohort management, and prompt configuration. It includes role-based access control with custom overrides, multi-level organization structures, and centralized prompt management for AI interactions. All admin pages require appropriate page:* permissions.

3 — Permission System
Multi-layered permission system: Role-based defaults (role_permissions), custom user overrides (user_permissions), admin wildcard access, resource-scoped permissions, expiring permissions, and middleware factory requirePermission() for route protection.

Role Permissions
  • role_permissions table stores default grants per role
  • Roles: admin, staff, builder, applicant, workshop_participant, workshop_admin, volunteer, enterprise_builder, enterprise_admin
  • Each permission has permission_key (e.g., page:admin_dashboard)
  • default_granted boolean determines default access
User Permissions
  • user_permissions table for custom overrides
  • Can grant or deny specific permissions per user
  • Supports resource-scoped permissions (resource_id, resource_type)
  • Optional expires_at for temporary access
  • Tracks granted_by and notes for audit

Organizations
  • Top-level entities with unique slug
  • Contact info, logo, active status
  • Can have multiple programs
Programs
  • Belong to an organization
  • program_type classification
  • Can have multiple courses
Courses
  • Belong to a program
  • level and sequence_order for ordering
  • duration_weeks for scheduling
Cohorts
  • Belong to a course
  • cohort_type: builder/workshop/external
  • access_code for enrollment
  • BigQuery integration for analytics
