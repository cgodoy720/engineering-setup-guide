Builder Onboarding System

system overview — wizard for admitted applicants

The Builder Onboarding system guides admitted applicants through required tasks before they become builders. It includes a task wizard, completion tracking, builder account creation, and email migration from personal to Pursuit email addresses. The system has both public-facing endpoints (for applicants) and admin endpoints (for managing tasks and email mappings).

Default Tasks: The system includes 8 default onboarding tasks: Program Details, Attendance Policy, Slack Setup, Pursuit Email, Google Calendar, Kisi Setup, Building in Public, and Engage Tech News. Tasks can be marked as required or optional, and admins can manage them via the admin API.
3 — Pages
Route Component Key Functionality
/onboarding OnboardingWizard Wizard interface for admitted applicants to complete required tasks before becoming builders. Shows all tasks with completion status, allows marking/unmarking tasks as complete, displays completion summary, and provides "Create Builder Account" button when all required tasks are done. Handles email migration (personal → pursuit email format: first.last@pursuit.org) and role change from applicant to builder.
4 — API Routes
Public Endpoints
  • GET /api/onboarding/tasks?applicantId= — Get all tasks with completion status for applicant
  • POST /api/onboarding/tasks/:taskId/complete — Mark a task as complete
  • DELETE /api/onboarding/tasks/:taskId/complete — Unmark a task (remove completion)
  • GET /api/onboarding/status/:applicantId — Get completion summary (required vs completed counts)
  • POST /api/onboarding/create-builder-account — Create builder account after all required tasks done
  • GET /api/onboarding/email-mapping/:email — Look up email mapping (personal → pursuit)
Admin Endpoints

All admin endpoints require authentication and appropriate permissions.

  • GET /api/onboarding/admin/tasks — List all tasks
  • POST /api/onboarding/admin/tasks — Create new task
  • PUT /api/onboarding/admin/tasks/:taskId — Update task
  • DELETE /api/onboarding/admin/tasks/:taskId — Delete task
  • GET /api/onboarding/admin/email-mappings — List email mappings
  • POST /api/onboarding/admin/email-mappings — Create email mapping
  • PUT /api/onboarding/admin/email-mappings/:id — Update email mapping
  • DELETE /api/onboarding/admin/email-mappings/:id — Delete email mapping
5 — Database Tables
3 tables support the onboarding system. See schema.sql for complete DDL with indexes and constraints.
onboarding_tasks
Configurable tasks with ordering, required flags, and active status. Default 8 tasks: Program Details, Attendance Policy, Slack Setup, Pursuit Email, Google Calendar, Kisi Setup, Building in Public, Engage Tech News.
  • task_id PK
  • title, description, detailed_description
  • link_url, link_text
  • is_required, display_order, is_active
  • created_by_user_id FK, updated_by_user_id FK
applicant_onboarding_task_completions
Tracks which tasks each applicant has completed. UNIQUE constraint on applicant_id + task_id ensures one completion record per task per applicant.
  • completion_id PK
  • applicant_id FK → applicant
  • task_id FK → onboarding_tasks
  • completed_at timestamp
  • notes text (optional)
applicant_email_mapping
Maps personal email addresses to Pursuit email addresses (format: first.last@pursuit.org). Both personal_email and pursuit_email have UNIQUE constraints.
  • mapping_id PK
  • personal_email UNIQUE
  • pursuit_email UNIQUE
  • created_at, updated_at
